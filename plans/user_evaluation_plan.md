# 🎙️ VO-SE Vocal 開発計画（2026年度）

## 1. プロジェクトビジョン
**「重音テトをはじめとする日本の歌声キャラクターを、最高の国産エンジンで歌わせる」**

- Synthesizer V のような「完璧に滑らかな歌声」ではなく、**キャラクターごとの「個性（クセ）」を最大限に活かす**エンジンを目指す。
- UTAU 音源と完全互換（ドラッグ&ドロップで追加可能）を維持しつつ、**公式キャラクターをブランド化**し、有料/無料のボイスバンクを展開する。
- **「生（ナマ）」モード**（UTAUライクな生々しさ）と **「艶（ツヤ）」モード**（滑らかで聞きやすい）をユーザーが選択可能にし、どちらの層も取り込む。

---

## 2. 優先度別タスク

### 🔴 優先度 1（即時対応：ビルド・起動の致命的バグ）

| 問題 | 原因 | 解決策 |
|------|------|--------|
| 公式音源がロードされない | `pack_all_voices.py` が空のヘッダを生成している | `assets/official_voices/` 以下の WAV を正しく `voice_data.h` に埋め込むようスクリプトを修正 |
| Windows で Core Engine が読み込めない | DLL パス解決が `sys._MEIPASS` に依存し、PyInstaller バンドルで失敗する | `get_engine_library_path()` を強化し、`os.add_dll_directory()` を Windows で呼び出す |
| 起動直後にクラッシュ（`rtmidi` 関連） | `vose_pro.spec` に `rtmidi` が明示されていない | `.spec` の `hiddenimports` に `rtmidi` と `mido.backends.rtmidi` を追加 |

---

### 🟠 優先度 2（ユーザー体験・競合優位性）

#### 2.1 Windows UI 崩れの修正
- **問題**：Mac 前提のスタイルシート、固定ピクセル幅、高DPI非対応。
- **対策**：
  - `main_window.py` の `apply_apple_refined_style()` を OS 分岐させ、Windows では `"Segoe UI"` / `Meiryo` を使用し、フォントサイズを 9pt に調整。
  - `keyboard_sidebar_widget.py` の `setFixedWidth(72)` を `setMinimumWidth(72)` + サイズポリシーに変更。
  - スクロールバーのハンドル高さを Windows では 30px に拡大。

#### 2.2 「生モード」と「艶モード」の実装
- **概要**：ユーザーが GUI 上で切り替えられる音質プリセットを提供。
- **技術的実装**：
  - `main_window.py` にコンボボックスを追加（`QComboBox`）。
  - 選択値を `vo_se_engine.py` に渡し、C++ エンジンの `execute_render()` の第4引数（`mode_flag`）を 0（生）または 1（艶）に設定。
  - **生モード**：WORLD の `q1` パラメータを 0.0 にし、`aperiodicity` をそのまま維持。AI ピッチ補正を弱める。
  - **艶モード**：`q1` を -0.15（従来）に戻し、ノイズ成分を抑え、AI 補正を強める。

#### 2.3 公式キャラのブランド化
- **ギャラリー表示**：公式音源を金色枠で囲み、最上部に固定表示する。
- **プレミアム感**：公式カードクリック時に「VO-SE 公式チューニング済み」バッジを表示。
- **音源管理**：公式音源は `assets/official_voices/` に ZIP 圧縮し、初回起動時に解凍する仕組みを導入（`voice_manager.py` に `extract_official_voices()` を追加）。

### 🟡 優先度 3（機能拡張・シンケンザーV との差別化）

| 機能 | 目的 | 実装方法 |
|------|------|----------|
| **Undo/Redo のピッチグラフ対応** | `GraphEditorWidget` の編集も履歴に残す | `GraphEditorWidget` にスナップショット機能を追加し、`HistoryManager` と連携 |
| **エフェクトチェーン GUI** | コンプ・リバーブ・EQ をノードエディタ風に操作 | `modules/gui/effects_panel.py` を新設し、DSP パラメータをスライダーで制御 |
| **VST/AU プラグイン対応** | DAW 内で VO-SE を使えるようにする | `Juce` や `iPlug2` を利用したラッパーを開発（中長期） |
| **AI モデルの多様化** | 「テトモード」「リアルモード」などプリセット追加 | ONNX モデルを複数用意し、ユーザーが選択できるようにする |

## 3. リリース・マーケティング戦略

- **バージョン 1.0**（2026年 Q3）：上記優先度1・2を全て修正し、公式音源「重音テト（仮）」を同梱。Windows/Mac/Linux で安定動作を確認。
- **バージョン 1.5**（2026年 Q4）：「艶モード」を標準搭載し、有料公式ボイスバンク（例：テト・エモーショナル）を販売開始。
- **コミュニティ**：Discord サーバーを開設し、ユーザーが作成した音源設定（OTO パラメータ）を共有できる「プリセットストア」を Web 上に構築。
