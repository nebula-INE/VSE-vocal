# VO-SE Pro ユーザー視点での評価計画

## 1. 評価項目

ユーザーが歌声合成ソフトウェアに期待する主要な要素に基づき、以下の項目でVO-SE Proを評価します。

- **操作性**: UI/UX、MIDI入力、歌詞入力、編集機能の使いやすさ。
- **音質・表現力**: 歌声の自然さ、表現の幅（ピッチ、ジェンダー、テンション、ブレスカーブ等）、AIの寄与。
- **主要機能**: 楽曲制作における基本機能（マルチトラック、タイムライン、グラフエディタ、再生・録音、Undo/Redoなど）の充実度。
- **拡張性**: 外部音源（UTAU）への対応、将来的な発展性。
- **安定性・パフォーマンス**: 動作の軽快さ、クラッシュ耐性、OS対応。
- **その他**: ライセンスモデル、コミュニティの可能性。

## 2. 既存情報からの評価構築

これまでの分析で得られた情報（`README.md`, `modules/audio/vo_se_engine.py`, `modules/gui/main_window.py`, `modules/gui/aural_engine.py`, `modules/audio/voice_manager.py`）に基づき、各項目で評価を具体的に記述します。

### 2.1. 操作性

- [`modules/gui/main_window.py`](modules/gui/main_window.py) の `apply_apple_refined_style()` メソッドに見られるように、AppleライクなミニマルなUIテーマが採用されており、視覚的に洗練された印象を与えます。これはユーザーにとって直感的で快適な操作体験に繋がる可能性があります。
- タイムラインウィジェットやグラフエディタウィジェット (`TimelineWidget`, `GraphEditorWidget`) がGUIの中心を担っており、ノートやピッチカーブの視覚的な編集が可能です。MIDI入力 (`load_midi_file`, `MidiInputManager`) や歌詞の一括入力 (`on_click_apply_lyrics_bulk`) 機能も実装されており、楽曲制作の主要なワークフローに対応しています。
- Undo/Redo機能 (`HistoryManager`) も整備されており、編集作業の安全性と効率性を高めます。

### 2.2. 音質・表現力

- `README.md` と [`modules/gui/aural_engine.py`](modules/gui/aural_engine.py) から、"Aural AI Engine" が "フルコンテキストラベル（アクセント・音素情報）を解析し、AIによる自然なピッチダイナミクスを生成する" との記述があり、自然な歌声表現へのAIの積極的な活用が期待できます。特に `AuralAIEngine` の `get_baked_pitch` や `generate_emotional_pitch` メソッドは、ピッチの "揺れ" を生成し、人間らしい歌声を実現する上で重要な役割を果たすでしょう。
- [`modules/audio/vo_se_engine.py`](modules/audio/vo_se_engine.py) では、ピッチ、ジェンダー、テンション、ブレスといった詳細なカーブ情報をC++エンジンに渡してレンダリングする "多重パラメーター・レンダリング" が実装されています。これにより、ユーザーは歌声のニュアンスを細かく調整し、高い表現力を引き出すことが可能です。
- C言語による高速信号処理の採用は、高品質な音質を維持しつつ、素早いレンダリングを実現し、制作のテンポを損なわないことに貢献すると考えられます。

### 2.3. 主要機能

- [`modules/gui/main_window.py`](modules/gui/main_window.py) に `VoseTrack` クラスが定義されており、"vocal"（歌声）と "wave"（オケ）のマルチトラックに対応しています。これにより、ユーザーは歌声トラックと伴奏トラックを組み合わせて楽曲を制作できます。
- タイムラインウィジェット (`TimelineWidget`) とグラフエディタウィジェット (`GraphEditorWidget`) は、ノート配置や各種パラメータカーブの編集を行うための中心的なGUI要素です。再生・停止・ループ機能 (`on_play_pause_toggled`, `stop_and_clear_playback`, `on_loop_button_toggled`) や、録音機能 (`toggle_recording`) も提供されており、DAWとしての基本的な機能を備えています。
- リアルタイムモニタリング機能 (`ProMonitoringUI`) は、レンダリングなしで "今あるデータ" を即座に合成して鳴らすことができ、ユーザーは調整した歌声の変化をリアルタイムで確認しながら作業を進めることができます。これは非常に強力な機能です。

### 2.4. 拡張性

- [`modules/audio/voice_manager.py`](modules/audio/voice_manager.py) では、`scan_voices` メソッドにより、**「VO-SE Official (Embedded)」として内部に公式音源を搭載**していることが確認できます。これにより、ユーザーはソフトウェアをインストールするだけで高品質な歌声合成をすぐに始めることができ、初期の利用ハードルが大幅に下がります。
- 加えて、標準的なUTAU音源パスと内部フォルダをスキャンし、`oto.ini` ファイルを解析することでUTAU音源の読み込みにも対応しています。これは、既存の膨大なUTAU音源資産をVO-SE Proで活用できることを意味し、ユーザーにとって非常に大きなメリットです。
- `README.md` と `main_window.py` の "AURAL FOUNDING VOICE PARTNERS" および "UNDER RECRUITMENT" という記述から、将来的なパートナー音源の追加も計画されていることが伺えます。公式音源と既存のUTAU音源、そして将来的なパートナー音源の組み合わせにより、VO-SE Proの音源ライブラリは今後さらに充実していく可能性を秘めており、長期的な利用価値と多様な表現の可能性を提供します。

### 2.5. 安定性・パフォーマンス

- `README.md` に "Python による高度な日本語言語解析と、C言語による高速な信号処理を組み合わせた、次世代ハイブリッド型音声合成プラットフォーム" との記述があるように、Pythonの柔軟性とC言語の高速性を組み合わせたハイブリッドアーキテクチャは、パフォーマンスと安定性の両立に貢献します。
- ONNX RuntimeによるAI最適化 (`aural_engine.py` の `_ort.InferenceSession`) は、AI処理の効率を高め、低スペック環境でも安定した動作を可能にします。
- WindowsとmacOSの両OSに対応しており、GitHub ActionsによるCI/CDパイプラインが構築されているため、ユーザーは安定したバイナリを継続的に受け取ることが期待できます。
- [`modules/gui/main_window.py`](modules/gui/main_window.py) の `_init_engines` メソッドにおけるMockクラスの導入は、"DLLやモジュールが欠落していても、Mockオブジェクトにより起動を阻止しません" とあり、開発中の堅牢性や、特定の環境でのエラー耐性を高めるための配慮がなされています。

### 2.6. その他

- [`modules/gui/main_window.py`](modules/gui/main_window.py) の `execute_export_pro_manager` メソッドや `SynthesisWorker` の `is_pro` フラグから、"Pro版" と "Free版" のライセンスモデルが存在し、Pro版ではより高いサンプリングレートやビット深度での出力が提供されることが伺えます。これにより、ユーザーは自身のニーズに合わせてソフトウェアを選択できます。
- パートナー募集に関する記述は、コミュニティとの連携や音源クリエイターとの協業を意識したものであり、ソフトウェアのエコシステムが発展していく可能性を秘めています。

## 3. 全体的なユーザー体験のまとめ

VO-SE Proは、DAWとしての基本的な操作性、Aural AIによる表現豊かな歌声合成、UTAU音源との互換性、そして安定したパフォーマンスを兼ね備えた、非常に有望な歌声合成ソフトウェアと言えます。特に、AIを用いた自然なピッチダイナミクス生成と、C言語による高速処理は、ユーザーがよりクリエイティブでストレスフリーな楽曲制作を行える環境を提供すると考えられます。今後の公式音源やパートナー音源の拡充により、さらに魅力的なプラットフォームとなるでしょう。

上記の評価計画について、ご意見や変更のご希望があればお知らせください。

## 💳 有料枠（Pro）で g_00074000 を使う

有料ユーザー向け機能を有効化するには、実行環境に以下を設定します。

- 環境変数: `VOSE_PLAN=pro`
- モデル配置: `models/g_00074000.onnx`（推奨）

現在の AI 推論エンジンは ONNX Runtime を使用するため、`g_00074000.pth` はそのままでは読み込めません。  
`.pth` を使う場合は ONNX へ変換するか、ONNX 版モデルを同梱してください。
